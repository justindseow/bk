INTRO

How to read this document

Each step shows you what the screen looks like, what happens when you click something, and what you see when you come back out.

Reading guide

  • Each major step has a main screen — this is what you see when you arrive at that step.
  • Where the BK needs to click into something (a split, a match, adding an entry), the document shows: the main screen with the row highlighted, the modal or panel that opens, and then the main screen after confirming.
  • At the end there is a feedback checklist. Please mark what works, what does not, and anything that seems missing.
  • All data shown is sample data for XYZ Co Sdn Bhd, January 2025. It is not real client data.
  • The tool is session-based. Each month you open it fresh, do the work, export the Journal Voucher, and close. Nothing carries over between sessions.
STEP 01

Document Collection

Before opening the tool, collect all source documents from the client for the period. The tool should not be started until documents are substantially complete.

Documents to collect before starting

  • Sales invoices — all invoices issued to customers
  • Purchase invoices / bills — from all suppliers
  • Payment vouchers — for all payments made, with supporting invoices attached
  • Receipts — for cash received without a formal invoice
  • Merchant statements — Grab Food, FoodPanda, or other platform settlements
  • Payroll summary — from HR or payroll system
  • Loan / HP statements — for any active financing facility
  • Bank statement — collected now but only loaded in Step 3 after all documents are posted

If documents are incomplete

  • Raise a query to the client before starting. List specifically which documents are missing.
  • Do not start posting until the client confirms either the documents are on their way or they authorise you to proceed with what is available.
  • Set the bank statement aside. It is not used until Step 3.

STEP 07

Handover Note — Next Session Checklist

At the end of every session, alongside the Journal Voucher, the tool generates a one-page Next Session Checklist. This is a briefing note from you to yourself for next month. It is pre-filled from what the tool already knows about this session.

Why this exists

  • The tool has no memory between sessions. Without this note, the BK has to remember accrual reversals, depreciation amounts, outstanding items, and carry-forward balances from memory.
  • At the start of next month, open this note before touching the tool. Work through the Must Do First section before posting any new documents.
  • The note is generated automatically — the BK does not need to fill it in. It is pre-populated from the session data.

STEP 08

Excel Download — What the BK Gets

At the end of the session, the BK downloads a single Excel file. It contains 8 tabs. Five are session outputs generated fresh from this month's work. Three are persistent files the BK keeps and brings to every session.

Tab structure

  • Journal Voucher — full debit/credit listing, grouped by section, source-tagged. The primary output for posting into the accounting software.
  • Trial Balance — auto-generated from the JV. All GL accounts with opening, movements, and closing balance. Foundation for management accounts.
  • Document Ledger (WP1) — all documents posted this session. Full audit trail of what came from which source document.
  • Bank Verification (WP2) — all bank rows with matching status. Includes reconciliation summary and timing items.
  • Adjusting Entries — all period-end entries with reversal flags. Reference if there is ever a question about what was accrued.
  • 💾 Next Session Checklist — persistent. The handover note. Keep this file.
  • 💾 Depreciation Schedule — persistent. Add a row each time an asset is capitalised. Reference every session.
  • 💾 Prepaid Schedule — persistent. Add a row each time a prepayment is posted. Reference every session.
